# Design Context — avoqado-superadmin

## Target audience

Personal interno senior de Avoqado: ops payments, KYC reviewers, ingeniería senior, founders. **Power users que viven en la herramienta 6+ horas/día**. No es público externo, no hay SEO. La velocidad de operación y la legibilidad bajo carga importan mucho más que el "wow" inicial.

## Use cases

- Revisar y aprobar KYC de venues nuevos
- Monitorear transacciones, identificar fallas, investigar disputas
- Configurar comisiones / pricing por venue / merchant accounts
- Lanzar updates de APKs a la flota de TPVs
- Audit log (quién hizo qué y cuándo, en qué venue, desde qué dispositivo)
- Administrar payment providers, aggregators, partner keys

## Brand personality / tone

**Editorial Operations Terminal.** Densidad informacional con cuidado tipográfico — inspiración: Stripe Dashboard, Linear, Mercury, Vercel internal tools. **NO** Bloomberg-monospace-everywhere; **NO** playful/toy; **NO** minimal-empty con whitespace gratuito; **NO** glassmorphic. Confiado, profesional, rápido, con un punto de vista tipográfico claro.

Avoqado es restaurant/payments tech en México — undertone cálido (canvas crema, no white puro; tinta tintada verde, no negro puro) pero la herramienta en sí es precisa y operativa.

## Aesthetic commitments

- **Tipografía**: Bricolage Grotesque Variable (display / KPI / headings — grotesca moderna distintiva con optical size axis) + Plus Jakarta Sans Variable (body / UI — refined, NO Inter) + JetBrains Mono solo para IDs/códigos (nunca como "vibe técnico")
- **Color**: paleta restringida en OKLCH, neutrales tintados hacia verde (`130°`), acento olivo `oklch(0.55 0.12 130)` aplicado con parsimonia. Sin gradientes purple-to-blue ni cyan-on-dark ni neón.
- **Espacio**: ritmo varia­do — secciones con respiración generosa, tablas con densidad alta. No "mismo padding everywhere".
- **Datos**: `font-variant-numeric: tabular-nums` para todo dato numérico/fecha. Alineación derecha para montos.
- **Status**: pills pequeñas con fondo tinte 8% + border 30% del color semántico (success / warn / danger / muted). NO círculos genéricos.
- **Componentes prohibidos** (AI slop fingerprints):
  - Sparklines como decoración (sólo si la información del trend es esencial y el lector la lee)
  - Cards anidadas / cards-para-todo (flatten hierarchy when possible)
  - Hero-metric template (número gigante + label diminuto + gradient accent)
  - Icon redondeado grande arriba de cada heading
  - Modal cuando hay alternativa (preferir drawer / inline expansion / nueva ruta)
  - Bounce / elastic easings

## Constraints

- **Idioma**: Español de México por default. Microcopy en español.
- **Framework**: React 18 + Vite + Tailwind v4 + shadcn-compatible (cuando shadcn primitive matche, OK; cuando no, escribir handcrafted).
- **A11y**: focus rings explícitos con `--ring`, todas las interactivas con `aria-label` donde el ícono lo amerite, tablas con `<th scope>`.
- **Timezone**: todo display de fecha pasa por `src/lib/datetime.ts`. Default `America/Mexico_City`; override con `venue.timezone` cuando aplica. Headers de tabla indican el TZ visible.
- **Permisos**: la UI asume rol `SUPERADMIN`. No hay variantes lite/full por rol dentro de este app.

## Differentiation — "lo único que se recuerda"

La sensación de que es un **terminal de operaciones con cuidado editorial**: la combinación de tipografía Bricolage en KPIs (poco común en B2B dashboards), neutrales cálidos tintados verde (no el gris frío de Linear / Vercel), y la densidad de Bloomberg pero sin su fealdad. El command palette (⌘K) es el acelerador principal — un power user nunca debería tener que tocar el sidebar después de la primera semana.
