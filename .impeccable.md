# Design Context — avoqado-superadmin

## Target audience

Personal interno senior de Avoqado: ops payments, KYC reviewers, ingeniería senior, founders. **Power users que viven en la herramienta 6+ horas/día**. No es público externo, no hay SEO. La velocidad de operación y la legibilidad bajo carga importan mucho más que el "wow" inicial.

## Use cases

- Revisar y aprobar KYC de venues nuevos
- Monitorear transacciones, identificar fallas, investigar disputas
- Configurar comisiones / pricing por venue / merchant accounts
- Lanzar updates de APKs a la flota de TPVs
- Audit log (quién hizo qué y cuándo, en qué venue, desde qué dispositivo)
- Administrar payment providers, aggregators, partner keys

## Brand personality / tone

**Minimal Tech Operations Tool.** Inspiración directa: GitHub Dark Mode. Limpio, directo, sin ornamento innecesario. La tipografía espeja la del dashboard principal (avoqado-web-dashboard) para consistencia cross-producto: Geist en el shell, Inter en el contenido, Geist Mono para datos. La personalidad viene de la paleta GitHub Dark y la densidad editorial, no de una fuente display propia. **NO** Bloomberg-monospace-everywhere; **NO** playful/toy; **NO** glassmorphic; **NO** purple-to-blue AI slop; **NO** corporate/enterprise vibes.

Avoqado es restaurant/payments tech en México — el dark theme usa la paleta GitHub Dark (`#0d1117` canvas). Botones primarios blancos con texto negro. El acento es **neutro / monocromo** (`#e6edf3`, ink/blanco) — nav selected, links, focus, brandmark; **el azul GitHub `#58a6ff` quedó reservado sólo para `--info`** (status). Badges pill-shaped sin bordes.

## Aesthetic commitments

- **Tipografía** (espejo de avoqado-web-dashboard): **Inter Variable** = área de contenido (default global `--font-sans`, así llega también a overlays portaleados a `document.body` — drawer, dialog, popover, command palette, toasts); **Geist Variable** = shell / sidebar / nav (override `--font-sans: var(--font-geist)` en el `<aside>`); **Geist Mono Variable** = IDs / versiones / numerales. La distinción tipográfica viene de sans-vs-mono + jerarquía de peso/tamaño, no de una fuente display separada. Sin Urbanist, sin JetBrains Mono.
  - **Vocabulario de tipos** (utilities en `src/app/index.css`, úsalos en vez de px inline): `eyebrow` = headers de sección / nav groups / headers de tabla; `label` = field labels (dt, label sobre input); `display` = headings/KPIs; `tabular` = numerales. Para pills/etiquetas (status, micro-tags tipo "Se encolará", indicadores tipo "Live") el primitive es el componente **`<Badge>`** (`size="sm"` o `md`), siempre pill sin borde. Todo en **sentence case** — sin `uppercase` ni `letter-spacing` editorial (se siente revista, no herramienta), salvo los dos wordmarks de marca (`Brandmark`, footer del command palette).
- **Color**: paleta GitHub Dark en hex. Canvas `#0d1117`, sunken `#010409`, raised `#161b22`. Ink `#e6edf3`. Borders `#21262d` / `#30363d`. Acento **neutro `#e6edf3`** (sin hue) usado con parsimonia (nav active, links, focus, brandmark). El azul `#58a6ff` quedó **sólo** para `--info` (status). Botón primario = `bg-white text-canvas`. Sin gradientes, sin neón, sin accent como color dominante.
- **Espacio**: ritmo varia­do — secciones con respiración generosa, tablas con densidad alta. No "mismo padding everywhere".
- **Datos**: `font-variant-numeric: tabular-nums` para todo dato numérico/fecha. Alineación derecha para montos.
- **Status / badges**: pills (`<Badge>`) con tint del color semántico, **sin borde**. El color codifica un **juicio**, nunca la identidad: `success` (verde) = saludable/positivo (Activo); `warn` (ámbar) = atención/transitorio (Pendiente, Mantenimiento); `danger` (rojo) = error/bloqueado; `info` (azul) = dato neutro que vale notar, con parsimonia; `muted` (gris) = sin juicio → **clasificaciones y tipos** (TPV Android, KDS, Inactivo, Retirado). Si un badge es un tipo/categoría y no un estado, va `muted` — colorear tipos es ruido. NO círculos genéricos.
- **Componentes prohibidos** (AI slop fingerprints):
  - Sparklines como decoración (sólo si la información del trend es esencial y el lector la lee)
  - Cards anidadas / cards-para-todo (flatten hierarchy when possible)
  - Hero-metric template (número gigante + label diminuto + gradient accent)
  - Icon redondeado grande arriba de cada heading
  - Modal cuando hay alternativa (preferir drawer / inline expansion / nueva ruta)
  - Bounce / elastic easings

## Constraints

- **Idioma**: Español de México por default. Microcopy en español.
- **Framework**: React 18 + Vite + Tailwind v4 + shadcn-compatible (cuando shadcn primitive matche, OK; cuando no, escribir handcrafted).
- **A11y**: focus rings explícitos con `--ring`, todas las interactivas con `aria-label` donde el ícono lo amerite, tablas con `<th scope>`.
- **Timezone**: todo display de fecha pasa por `src/lib/datetime.ts`. Default `America/Mexico_City`; override con `venue.timezone` cuando aplica. Headers de tabla indican el TZ visible.
- **Permisos**: la UI asume rol `SUPERADMIN`. No hay variantes lite/full por rol dentro de este app.

## Differentiation — "lo único que se recuerda"

La sensación de que es una **herramienta personal, no un producto corporativo**: la tipografía consistente con el dashboard principal (Geist + Inter + Geist Mono) hace que se sienta parte de la misma familia de producto, la paleta GitHub Dark con botones blancos se siente familiar y funcional, y los badges pill-shaped sin bordes se sienten limpios. El command palette (⌘K) es el acelerador principal — un power user nunca debería tener que tocar el sidebar después de la primera semana.

## Design Principles

Cinco lentes operativos. Si una decisión de diseño no se puede defender contra estos, está mal. Síntesis del manifesto anterior — léelos primero, baja al detalle sólo si el caso lo requiere.

1. **Operative speed beats first impression.** El operador vive aquí 6+ h/día. Densidad tipográfica, atajos de teclado y `⌘K` ganan sobre hero animations y wow moments. Touch targets ≥ 36 px porque ops también revisa desde móvil; el sidebar fijo desaparece bajo `md` y se reemplaza por drawer.
2. **Editorial density, never Bloomberg ugliness.** La tipografía conduce el layout. Misma familia que el dashboard principal (Geist shell + Inter contenido + Geist Mono datos) — la jerarquía se logra con peso/tamaño y el contraste sans-vs-mono, no con una fuente display extra. Ritmo variado: secciones respiran, tablas compactan; "mismo padding everywhere" es flat e impersonal.
3. **GitHub Dark, not corporate.** Paleta GitHub Dark (`#0d1117` canvas, acento **neutro `#e6edf3`** sin hue; el azul `#58a6ff` sólo como `--info`). Botón primario = blanco con texto negro. Badges pill-shaped sin bordes. Prohibidos: gradient purple-to-blue, cyan-on-dark, neón, glassmorphism, indigo/purple accents, dark mode "azul Discord". Se siente como herramienta personal, no como producto enterprise.
4. **Numerals are data, not decoration.** `font-variant-numeric: tabular-nums` siempre; montos alineados a la derecha; status pills semánticas (tint 8 % + border 30 %, no círculos genéricos). Las sparklines aparecen sólo si el trend es información esencial que el lector va a leer — nunca como garnish.
5. **Empty states teach the interface.** "No hay KYC pendientes. Última revisión hace 3 h" en vez de "Nothing here". Cada vacío explica qué hace la pantalla, qué espera, y qué pasó por última vez. El estado vacío también es UI.
